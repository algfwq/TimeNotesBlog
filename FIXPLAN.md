TimeNotesBlog 修复与重构计划
已确认的产品决策
采用完整分期交付，按风险优先：先安全和功能硬伤，再阅读器/封面，最后公开端与后台视觉重构。
主题支持浅色/深色，初始跟随 prefers-color-scheme，用户在导航切换后以 Cookie 覆盖系统偏好。
首页 Hero 的背景、标题和副标题由管理员后台统一配置；背景支持上传或 URL；滚动越过 Hero 后展示手账网格。
Blog 阅读器直接复制客户端只读阅读器并后续手工同步，而不是维护当前简化版；上传用户视为可信，保留 .tnote 原始富文本 HTML。
无 thumbnail.png 的手账必须先通过 TimeNotes 客户端生成并写入缩略图后才能上传；后台直接上传会被服务端拒绝并引导至客户端。
后台编辑通过受保护的本机 TimeNotes loopback 配对桥打开和回传更新，不新增浏览器编辑器。
默认 admin 第一次进入后台必须改用户名和密码；永远保留至少一个管理员；删除用户前将其手账事务性转移给指定管理员。
PoW 改为可用的轻量 bit 难度方案（会话/IP 绑定、短时单次、Web Worker、失败递增、限流）。
公开端与后台均通过 WebSocket 接收实时增量；断线重连后重新获取快照。
下载授权经 WebSocket 获得短时、一次性令牌，完整 .tnote 由同源 HTTP 流式传输。
每本笔记由管理员单独设置是否允许公开下载；后台始终允许管理员下载。
阶段 A：可靠性、安全与可用管理基础
A1. 静态资源与通信基础
将 Vite 输出资源从相对路径调整为根路径可访问的绝对路径，验证 /note/:id 与 /admin/{token}/... 下 JS/CSS/字体不会被 SPA fallback 返回为 HTML。
扩展 BlogWS：增加认证状态、订阅/取消订阅、事件监听、自动重连和重连后的快照回调；保留请求/响应关联 ID。
Hub 增加权限感知的会话注册、事件广播和慢客户端隔离：锁内只复制会话，锁外发送；队列满时只断开慢会话。
新增公开和管理员事件类型：笔记、互动、评论、封面、站点设置、用户权限和统计变化。客户端按资源 ID 合并增量；无法安全合并或版本跳变时全量刷新。
A2. 真实客户端 IP、GeoIP 与隐私
将 Fiber WebSocket 升级阶段由 clientIP 计算出的真实 IP 传递给 WebSocket session，避免 conn.IP() 覆盖可信代理解析结果。
仅当直连端属于 trustedProxies 时才使用 X-Forwarded-For；忽略非可信来源伪造头。
让点赞去重、登录失败计数、PoW、评论限流、访问统计和 GeoIP 均使用统一的 IP 哈希。
跳过回环、私有、链路本地和未指定地址的地理解析和地图统计；既不向外部 GeoIP 查询，也不产生地图点。
GeoIP 异步查询成功后回填触发该查询的访问记录，并广播更新后的管理员统计，而不是等到同一 IP 下一次访问。
日志不记录完整 IP、密码、JWT、PoW nonce、评论正文或 .tnote 原始数据。
A3. PoW 与认证
将 PoW 难度语义从 SHA-256 十六进制前导零字符改为前导零 bit；配置改为可验证的 base/max bit 范围。
Challenge 记录绑定的 IP 哈希和临时会话标识、签发时间、过期时间和单次使用状态；验证后无论成功失败均消费。
对 challenge 签发和密码登录分别限流；挑战和按 IP limiter 状态均实现 TTL 清理，避免内存持续增长。
前端和 TimeNotes 客户端将 PoW 放进 Web Worker，显示求解状态并设定合理超时；失败次数带来有限的小幅难度提升，始终可在普通浏览器几秒内完成。
JWT 解析后回查数据库用户的存在、角色、上传权限和 must_change_credentials 状态；被删除、降级或禁用后立即失效，不等待 JWT 自然过期。
A4. 默认管理员、权限与所有权
迁移新增 must_change_credentials，仅首次自动创建的 admin / 123456 标记为真。
登录/session 响应带回该标志。后台登录后展示不可关闭的强制模态框，只有同时提交有效且不重名的新用户名和新密码才可关闭并进入后台。
用户管理扩展为：用户名、密码重置、角色、上传权限、创建时间和更新状态；管理员对包含原始 HTML 上传信任边界给予明确提示。
增加存储层的管理员数量检查。禁止删除自己、删除最后管理员、把最后管理员降级为 user；后端执行业务不变量，前端只负责提前禁用和解释。
删除用户的请求必须携带目标管理员 ID。SQLite 事务内验证目标、将其所有笔记转移、删除用户并触发用户/笔记实时事件。
管理员自身账号设置独立于用户列表，调用现有 self-update 路径并完整更新本地会话展示。
A5. 上传、归档验证、缩略图与下载
将上传状态改为受互斥保护的顺序状态机：上传所属用户、连续 chunk index、每块/累计尺寸、声明大小和 SHA-256 都必须匹配；结束、断连或过期后清理 .part 文件和内存状态。
新增纯 Go .tnote 归档校验器，复用客户端 safeArchiveName 的安全语义：拒绝绝对路径、..、重复/非法条目、压缩炸弹、过大条目和超量条目；检查 ZIP、manifest.json、document.json 与支持的格式版本。
要求上传归档含根目录真实 PNG 的 thumbnail.png，并限制文件尺寸和图像像素。后台浏览器直传无有效缩略图时，返回结构化错误，提示在 TimeNotes 客户端打开并重新上传。
Blog 从通过验证的 ZIP 安全抽取封面到受控 cover 目录，以笔记 ID 管理；数据库 notes 增加封面引用、公共下载开关及必要版本/更新时间字段。
创建/更新采用原子流程：先验证临时 .tnote 和封面 → 移入新文件与 cover → 事务更新数据库 → 清理旧文件；任何步骤失败不改变旧笔记。
提供只返回已验证封面的受控静态端点，不需要公开下载整个 ZIP 来渲染首页。
下载令牌改为一次性、短时、按权限签发和消费后删除；管理员总可下载，公开访客仅对 public_download 为 true 的可见笔记获得令牌。HTTP /files/:token 只负责流式发送，不承担业务授权。
A6. 评论身份、互动和编辑桥
评论 SideSheet 删除常驻身份三输入框。点击发布时读取 tn_blog_comment_identity Cookie；有有效身份则提交，无身份显示 Semi 模态框。
身份模态框要求昵称，且邮箱/GitHub URL 至少一个；保存 Cookie 后自动继续原评论提交，保留评论正文。后端继续独立校验格式、长度、可见性与 IP 限流。
维持点赞不可取消、每 IP 每笔记一次；互动更新立即广播给相关公开页和后台。
改造 TimeNotes blog_bridge.go：仅监听 loopback，取消反射任意 Origin；引入配对流程、短时一次性能力令牌、笔记 ID/下载 URL 白名单、目录限制与过期校验。
后台编辑动作先向 Blog 获取已授权笔记下载能力，再与本机 TimeNotes bridge 配对并交付单笔记能力。客户端仅将该授权文件保存至其 notebooks 目录，打开后保留远端笔记关联信息；保存时对同一笔记执行更新上传。
对客户端未运行、未配对、过期、拒绝写入和版本冲突提供明确错误与手动下载回退。
阶段 B：封面生产与客户端阅读器复用
B1. TimeNotes 客户端封面持久化
以 HomeWorkspace 现有 html2canvas 首页面预览为基础，补齐可持久化封面生成链：首页面预览渲染 → canvas → 真正 PNG bytes/data URL → NotePackage.Thumbnail → 写入 ZIP 根 thumbnail.png。
保留用户已经显式设置且有效的封面；无封面才生成默认封面。
修复现有缩略图写入的 MIME 不一致：即使源图为 JPEG/WebP，也在写入 thumbnail.png 前转为真实 PNG；读取端不再无条件把任意字节标为 image/png。
将已有缩略图贯穿普通保存、编辑后保存、Blog 上传和 Blog 更新，避免 createPackage 目前传空字符串导致封面丢失。
客户端上传前确保缩略图存在，然后上传完整、包含封面的 .tnote。Blog 不在 Go 服务端模拟 DOM 截图。
B2. Blog 阅读器迁移
以客户端 ReadOnlyViewer.tsx 为源，复制双页书本、翻页动画、缩放、拖拽、键盘导航和工具栏至 Blog，替换当前简化 ReaderView。
复制必要的只读渲染依赖并适配 Blog 环境：页面背景和裁剪、字体注入、富文本、绘图、胶带、图片/贴纸、音频、视频、代码块与 GLB 模型渲染。
将 Blog 的 tnote.ts 扩展到客户端文档类型与资源组；资源使用 Blob URL 而非将全部大资源转为 data URL。路由退出和文档替换时统一释放 Object URL。
保留上传者可信的富文本策略：按原 HTML 渲染；外部链接始终经安全的新窗口处理。
阅读器外壳加入点赞、评论 SideSheet，以及“下载”按钮（仅该笔记 public_download 启用时展示）。互动不替代阅读器本身。
B3. 公开手账展示
笔记卡片使用服务器封面端点，展示封面、标题、上传者、更新时间、点赞与评论数量及可选公开下载标记。
现有封面一律直接呈现；无效或缺失封面的记录显示明确损坏/待修复状态，而非将截图伪装成阅读功能。后台提供重新经客户端上传的操作说明。
公开端收到笔记/封面/互动事件后按 ID 更新；阅读页收到当前笔记隐藏/删除事件时显示状态并返回首页。
阶段 C：主题、公开首页、后台重构与统计地图
C1. 主题系统与公开首页
将现有全局玻璃拟态、暗色硬编码和 Vite 模板残留替换为以 Semi ConfigProvider token 为主的浅/深主题；只保留组件无法表达时必要的布局样式。
ThemeProvider 启动时读取 Cookie；没有显式偏好则监听 prefers-color-scheme。顶部导航主题按钮写 Cookie 并立刻更新 Semi、阅读器、图表和页面背景。
管理员可维护 site_settings：Hero 标题、副标题、背景模式（上传/URL）、背景资源/URL、位置焦点、遮罩颜色和透明度。上传图片验证实际 MIME、尺寸、像素和存储路径；拒绝 SVG/HTML 伪装图片。
公开首页为全高 Hero，移除搜索框及其下方链接；页面下滚进入客户端文件管理器语义的封面网格。保存设置后经 WebSocket 向所有打开的首页广播并即时更新。
C2. Semi 后台
用 Semi Layout、Nav、Card、Table、Form、Modal、Drawer、Upload、Tag、Empty 和响应式栅格重写后台，统一支持浅/深主题。
仪表盘展示今日/近期访问、手账总量、互动摘要、趋势和地理地图；所有数据由后台事件实时更新。
手账管理提供缩略图、元数据、上传、下载、编辑到客户端、公开/隐藏、单笔记公开下载开关、删除；无手动刷新依赖。
用户管理提供完整编辑、管理员/上传权限、强约束提示及转移删除流程；新增当前账号安全页。
站点外观页提供背景上传/URL、标题/副标题、遮罩和焦点预览，发布后实时更新公开首页。
C3. 世界地图
引入本地打包世界 GeoJSON，并配置 VChart geographic/Geo 图层；不依赖第三方地图服务和 API key。
后端按国家/地区地理聚合访问量，私有/回环/无坐标地址不进入结果。
地图按访问次数分段/连续色阶着色，数值越高颜色越深；tooltip 显示国家、地区、城市和次数。保留 14 日趋势和手账互动表。
